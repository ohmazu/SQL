# 1️⃣ 개념 정리

## 01. 여러 가지 과제 

### 빅데이터 분석 환경의 변화
- 기업이 빅데이터 분석 기반을 도입하면 
    - 서비스 -> 데이터베이스 -> 데이터 -> 리포트 -> 경영층
- 의 과정이 생김
<br>
- 데이터 분석을 위해서는
    - 데이터를 한 곳에 저장
    - SQL을 통해 원하는 데이터를 추출
- 해야 함

### 분석 담당자의 과제
- 기존 환경
    - 도구가 제공하는 지표 사용
    - 마우스 클릭으로 데이터 추출 

- 빅데이터 환경
    - 스스로 지표 정의 + SQL로 데이터 추출 
- 발생하는 문제
    - 지표를 직접 정의
    - 데이터 추출을 위해 SQL 필요

### 해결 방향
- 분석 담당자, 엔지니어 모두 역량 확장 필요
- 분석 담당자
    - SQL 학습
    - 데이터 가공
    - 분석 방법 이해
- 엔지니어
    - 분석 SQL 이해
    - 리포트 구조 이해
    - 데이터 활용 방식 이해

### 현업에서는...?
**1. 데이터 분석**
- 데이터 추출
    - 특정 기간 구매 고객
    - 특정 서비스 이용 기록 
    - 특정 행동 이벤트 
- 데이터 집계
    - 카테고리별 매출
    - 채널별 유입 수
- 테이블 결합(JOIN)
- 분석용 파생 변수 생성 
- 분석용 데이터셋 생성 
-> 이후에 Python 등으로 분석

**2. CRM**
- 고객 세그먼트 추출
- 마케팅 타겟 추출
- 캠페인 성과 분석
- 고객 행동 분석

## 02. 시스템
대표적인 시스템
1. PostgreSQL
2. Apache Hive
3. Amazon Redshift
4. Google BigQuery
5. SparkSQL
- 대부분 SQL 인터페이스를 재공해서 SQL로 데이터 처리 가능

### PostgreSQL
- 개념
    - 오픈소스 관계형 데이터베이스
- 특징
    - 표준 SQL 준수
    - 윈도우 함수, CTE 등 분석용 SQL 지원
    - GUI 설치 가능
    - 로컬에서도 사용 가능
- 사용 목적 
    - 소규모 데이터 분석
    - SQL 학습
    - 일반적인 서비스 DB
- 한계
    - 대규모 데이터 처리에는 성능 한계 존재

### Apache Hive
- 개념
    - Hive = HDFS + SQL 인터페이스
    - 분산 파일 시스템 데이터를 SQL로 처리
- 구조
    - SQL -> 맵리듀스 작업으로 변환 -> 병렬 처리
- 특징
    - 대용량 데이터 처리 가능
    - SQL처럼 사용 가능
    - UDF 지원 
- 단점
    - 실행 속도 느림
    - 실시간 분석에는 부적합
### Amazon Redshift
- 개념
    - AWS에서 제공하는 분산 병렬 데이터베이스
- 특징
    - 진짜 RDB 구조라 UPDATE, DELETE, 트랜젝션 지원 가능 
- 장점
    - 매우 빠른 쿼리 속도
    - 클라우드 기반
    - 쉽게 확장 가능
- 단점
    - 사용 비용 발생
    - 성능 튜닝 필요
    - 전문가 필요

### Google BigQuery
- 개념
    - 구글이 제공하는 클라우드 기반 빅데이터 분석 시스템
- 특징
    - CTE 지원
    - 서브쿼리 지원
    - 일반 SQL과 유사
- 장점
    - 서버 관리 필요 없음
    - 구글 서비스와 연동 쉬움
    - 작은 데이터면 저비용 
- 단점
    - 쿼리 실행 시 읽은 데이터 양 기준 비용 발생

### SparkSQL
- 개념
    - Apache Spark에서 제공하는 SQL 기반 데이터 처리 가능
- 특징
    - 매우 빠른 분산 처리
    - 다양한 언어 지원
    - 머신러닝까지 가능 
- 장점 
    - 하나로 데이터 처리, SQL 분석, 머신러닝, 그래프 처리, 스트리밍 가능

### Summary

| 시스템        | 특징       | 사용 목적       |
| ---------- | -------- | ----------- |
| PostgreSQL | 전통 RDB   | 소규모 데이터     |
| Hive       | 분산 파일 기반 | 초대용량 데이터    |
| Redshift   | 클라우드 DW  | 빠른 분석       |
| BigQuery   | 서버리스 분석  | 대용량 분석      |
| SparkSQL   | 분산 처리 엔진 | 데이터 처리 + ML |


## 03. 데이터 
```
✅ 학습 목차
* 데이터의 종류
* 업무 데이터
* 로그 데이터
* 두 데이터를 사용해서 생성되는 가치
```

### 업무 데이터 (Business Data)
- 서비스 운영을 위해 DB에 저장되는 데이터 
- 특징
    - 데이터 정밀도가 높음
    - 데이터가 갱신됨 
    - 여러 테이블을 JOIN 해야 함
- 내부 구조
    - 트랜잭션 데이터
        - 사용자의 행동 기록
    - 마스터 데이터 
        - 서비스의 기준 정보

### 로그 데이터 (Log Data)
- 사용자의 행동을 기록한 데이터
- 특징
    - 누적 데이터
    - 데이터 정확도 낮음
    - 행동 분석에 적합 

| 구분     | 업무 데이터 | 로그 데이터    |
| ------ | ------ | --------- |
| 목적     | 서비스 운영 | 사용자 행동 기록 |
| 정확도    | 높음     | 낮음        |
| 데이터 변경 | 수정 가능  | 수정 없음     |
| 분석     | 매출, 주문 | 방문, 클릭    |
| 구조     | RDB    | 이벤트 로그    |

### 두 데이터를 함께 쓰는 이유 
- 로그 데이터를 이용하면 어떤 페이지를 방문했는지느 알지만
    - 구매했는지
    - 매출 얼마인지
- 는 알 수 없음
- 따라서 로그데이터 + 업무 데이터를 같이 분석함
-> 퍼널 분석
